Use of Virtual Environments in Python Projects:

  • Virtual Environment is used to seperate out the versions of packages for eg., Django or Flask.
  • In general, if we install Packages directly without Virtual Environment then they will be installed on global levels
  • If you have multiple Django projects where some older projects need previous versions of Django and when you need a new project with latest version of Django package and if you install it without virtual env then your older projects would be affected since the older version of Django package will be replaced with newer version after the upgrade.
  • Hence it is best practice to use Virtual Env for each project and have whatever packages needed for that particular project inside the virtual environment.
  • venv is recommended for creating Virtual Environments from Python 3.5
  • Create a folder for the new project then cd to the folder and execute below to create new virtual env

Creating Virtual Env:

In Windows: python -m venv env_name

In Linux: python3 -m venv env_name (assuming you have python3 version in your machine)

Now, 'env_name' will be created inside the project folder

To acitvate the Virtual Env:

In Windows: env_name\scripts\activate.bat

In Linux: source env_name\bin\activate.bat

After activation: (venv) C:\Users\Name\Folder>

Do not store any project scripts inside 'env_name' folder

pip list -- to get list of packages installed in the current venv

pip freeze -- to get list of packages in a format usable to copy for requirements.txt

Below commands a requirements.txt file with all packages installed in the project pip freeze > requirements.txt

To directly install all packages from requirements.txt use pip install -r requirements.txt
